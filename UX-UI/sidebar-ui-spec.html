8. Header / Top Bar

Design choice: MESA ERM non ha un header/top bar tradizionale. Il layout e' "sidebar-only" — la zona superiore del sidebar (Logo + Quick Nav) funge da header. Il content area occupa tutto lo spazio restante senza alcuna barra orizzontale in cima.

8.1 Come Replicare con un Header Classico

10. Mobile & Responsive

Stato attuale: MESA ERM non implementa gestione mobile/responsive. Non ci sono breakpoint, media query, useBreakpoint, ne' Drawer per la sidebar su mobile. Il layout e' progettato esclusivamente per desktop (1280px+).

10.1 Raccomandazioni per il Mobile

Per replicare il design MESA con supporto mobile, ecco le specifiche raccomandate:

Codice React raccomandato

import { Grid, Drawer } from 'antd';
const { useBreakpoint } = Grid;

function AppShell() {
  const screens = useBreakpoint();
  const isMobile = !screens.md;   // < 768px
  const isTablet = screens.md && !screens.lg; // 768-1023px
  const [drawerOpen, setDrawerOpen] = useState(false);

  if (isMobile) {
    return (
      <Layout>
        <MobileHeader onMenuClick={() => setDrawerOpen(true)} />
        <Drawer
          placement="left"
          width={280}
          open={drawerOpen}
          onClose={() => setDrawerOpen(false)}
          styles={{ body: { padding: 0 } }}
        >
          <SidebarContent onNavigate={() => setDrawerOpen(false)} />
        </Drawer>
        <Content style={{ marginTop: 48 }} />
      </Layout>
    );
  }

  return (
    <Layout style={{ height: '100vh' }}>
      <Sider
        width={240}
        collapsedWidth={48}
        collapsed={isTablet || sidebarCollapsed}
      >
        <SidebarContent />
      </Sider>
      <Content />
    </Layout>
  );
}

}
